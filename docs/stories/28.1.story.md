### Content Guidelines

#### CONTRIBUTING.md Should Include:

1. **Development Environment Setup**
   - Python 3.11+ requirement
   - Virtual environment setup: `python -m venv venv`
   - Activate venv: `source venv/bin/activate` (Mac/Linux) or `venv\Scripts\activate` (Windows)
   - Install dependencies: `pip install -r requirements.txt`
   - Install dev dependencies: `pip install -r requirements-dev.txt`
   - Environment variables: Copy .env.example to .env, add CFBD_API_KEY

2. **Running the Application**
   - Backend: `python main.py` (or `python -m uvicorn main:app --reload`)
   - Frontend: Open browser to `http://localhost:8000/frontend/`
   - API docs: `http://localhost:8000/docs`

3. **Running Tests**
   - All tests: `pytest -v`
   - Unit tests only: `pytest -m unit -v`
   - Integration tests: `pytest -m integration -v`
   - E2E tests: `pytest -m e2e -v`
   - With coverage: `pytest --cov=. --cov-report=html`

4. **Code Contribution Workflow**
   - Fork and clone repository
   - Create feature branch: `git checkout -b feature/description`
   - Make changes and commit: `git commit -m "Description"`
   - Push to fork: `git push origin feature/description`
   - Open pull request to main branch

5. **Code Review Checklist**
   - All tests pass locally
   - Code follows PEP 8 style guide
   - New features have tests
   - Documentation updated if needed
   - No sensitive data (API keys, credentials) committed

[Source: `docs/TESTING.md`, `docs/CI-CD-PIPELINE.md`]

#### DEVELOPMENT.md Should Include:

1. **High-Level Architecture Overview**
   - System context diagram reference (see architecture.md)
   - FastAPI backend + SQLAlchemy ORM + SQLite
   - Vanilla JavaScript frontend (no frameworks)
   - Nginx reverse proxy in production
   - CollegeFootballData API integration

2. **Core Module Responsibilities**
   - `main.py` (450 lines) - FastAPI app, all API endpoints, middleware
   - `ranking_service.py` (363 lines) - Modified ELO algorithm, business logic
   - `models.py` (146 lines) - SQLAlchemy ORM models (Team, Game, RankingHistory, Season, Prediction)
   - `schemas.py` (200 lines) - Pydantic validation schemas for API
   - `database.py` (48 lines) - DB connection, session management
   - `cfbd_client.py` (150 lines) - External API client

3. **Key Design Patterns**
   - **Dependency Injection**: FastAPI's `Depends()` for DB sessions
   - **Service Layer**: `ranking_service.py` separates business logic from API
   - **ORM Pattern**: SQLAlchemy models with relationships
   - **Repository Pattern**: Models encapsulate data access
   - **Pydantic Validation**: Automatic request/response validation

4. **Common Development Tasks**
   - Adding a new API endpoint: Add route in main.py, create schema in schemas.py
   - Modifying ELO algorithm: Edit ranking_service.py methods
   - Adding a database field: Update models.py, create migration script
   - Adding tests: Create test file in tests/, use existing fixtures
   - Importing data: Use `import_real_data.py` with CFBD_API_KEY

5. **Troubleshooting Guide**
   - **Import errors**: Check virtual environment is activated
   - **Database errors**: Delete cfb_rankings.db and run seed_data.py
   - **API call failures**: Verify CFBD_API_KEY in .env
   - **Test failures**: Check pytest markers, ensure fixtures loaded
   - **Frontend 404s**: Ensure backend running, check proxy config
